Canopy Structure & Biomass (Prescribed)

This page documents the prescribed canopy structure/biomass component used by ClimaLand’s vegetation standalone runs. This module does not prognose vegetation; it simply exposes structural and biomass properties to other sub‑models (radiative transfer, stomata/photosynthesis, energy, hydraulics) as given inputs.

It corresponds to src/standalone/Vegetation/biomass.jl.

Summary

  • Purpose: provide fixed canopy structural attributes and biomass pools needed by the vegetation modules.
  • Dynamics: none in this standalone component (no growth/turnover). Values are prescribed at initialization (and optionally via time‑varying drivers).
  • Consumers: canopy radiative transfer, canopy energy balance, stomatal/photosynthesis, hydraulics/soil‑root coupling.

What it contains (typical fields)

Exact field names/types are defined in biomass.jl. The list below reflects the kinds of quantities exposed; match to your codebase when wiring configs/diagnostics.

  • Structural state

    • Total LAI and its partition into sunlit/shaded (via RT), and optionally vertical leaf area density if used by your RT.
    • Canopy height, displacement height, roughness length (if provided from structure here rather than the micromet module).
    • Clumping/leaf angle parameters (used by RT if applicable).

  • Biomass pools (per ground area)

    • Leaf biomass (kg m⁻²)
    • Stem/wood biomass (kg m⁻²)
    • (optional) Fine root biomass (kg m⁻²)
    • (optional) Storage/carbohydrate pools (if routed to respiration/phenology elsewhere)

  • Leaf/plant traits (static)

    • Specific leaf area (SLA) or its inverse LMA for LAI↔biomass conversions.
    • Leaf optical properties pointers (often configured in the RT module but may be colocated).
    • (optional) Nitrogen content/traits if other modules need them.

How values are provided

  • Prescribed constants: Most commonly, values are specified directly in the configuration (e.g., a Config/Struct with LAI, height, biomass values).
  • Time‑varying drivers (optional): You may supply external forcings (e.g., diurnal/seasonal LAI from a file) and map them to this component at each timestep. If used, this module simply reads the current value—no internal prognostic update is performed.
  • Consistency checks: If both LAI and leaf biomass are provided alongside SLA/LMA, the initialization can check consistency (LAI ≈ biomass × SLA).

Using the prescribed canopy structure in a standalone run

using ClimaLand

canopy_struct = Vegetation.Biomass.Config(
    LAI = 3.0,              # m2 leaf / m2 ground
    canopy_height = 12.0,   # m
    leaf_biomass = 0.3,     # kg m^-2
    wood_biomass = 10.0,    # kg m^-2
    SLA = 10.0,             # m2 kg^-1  (example)
    # ... any other fields defined in biomass.jl
)

veg = Vegetation.CanopyConfig(
    biomass = canopy_struct,
    # other vegetation submodules (RT, stomata, energy, etc.)
)

model = ClimaLand.Standalone.CanopyModel(vegetation = veg)
sol = ClimaLand.solve!(model; tspan = (0.0, 86400.0))

Replace field names with those in your version of biomass.jl (e.g., LMA instead of SLA, h_canopy, etc.).

Diagnostics

Typical diagnostics exposed to other modules (names may vary):

  • :LAI, :LAI_sun, :LAI_shade — total and partitioned LAI (m² m⁻²)
  • :canopy_height — canopy top height (m)
  • :leaf_biomass, :wood_biomass, :root_biomass — pools (kg m⁻²)
  • :SLA or :LMA — trait used for LAI↔biomass conversion
  • (if time‑varying drivers) the current values are echoed each step

Tips

  • Keep units consistent across modules (e.g., LAI in m² m⁻², biomass in kg m⁻², height in m).
  • If coupling to RT that expects vertical structure, provide a leaf area density profile or enable an internal splitter consistent with your LAI and canopy height.
  • When using time‑varying LAI, coordinate with stomatal/photosynthesis parameterizations that scale per‑leaf vs per‑ground to avoid double‑counting.
  • Ensure roughness/displacement parameters used in micrometeorology match the prescribed canopy height and structure.
